New installs get Telegram automatically (the installer handles everything). This doc is for adding Telegram to an **existing deployment** that was installed before this change.

## What changed

- **fagents-cli** — new repo. Contains `telegram.sh`, a bash+curl+jq Bot API client. Commands: `whoami`, `send <chat-id> <message>`, `poll`.
- **fagents-autonomy** — `collect_telegram()` in `daemon.sh` now calls `telegram.sh poll` via sudo and writes messages to the inbox queue. Was previously a stub returning 1.
- **fagents** — installer creates credential store, sudoers rules, clones fagents-cli.

## Prerequisites

- One Telegram bot token per agent (from [@BotFather](https://t.me/BotFather))
- Each agent gets their own bot — `getUpdates` is destructive (consumes offsets), so only one poller per bot

## Setup steps

All commands run as root on the fagents host.

### 1. Clone fagents-cli

```bash
INFRA_USER="fagents"
INFRA_HOME=$(eval echo "~$INFRA_USER")

# Bare repo
su - "$INFRA_USER" -c "git clone --bare https://github.com/fagents/fagents-cli.git ~/repos/fagents-cli.git"
su - "$INFRA_USER" -c "git -C ~/repos/fagents-cli.git remote remove origin"
chmod -R g+rX "$INFRA_HOME/repos/fagents-cli.git"

# Working copy
su - "$INFRA_USER" -c "git clone $INFRA_HOME/repos/fagents-cli.git ~/workspace/fagents-cli"
chmod -R g+rX "$INFRA_HOME/workspace/fagents-cli"
```

### 2. Update fagents-autonomy

Pull the latest `daemon.sh` which has the real `collect_telegram()`:

```bash
su - "$INFRA_USER" -c "cd ~/workspace/fagents-autonomy && git pull"
```

Each agent also needs the updated daemon in their workspace:

```bash
for user in coo dev; do  # adjust to your agent users
    su - "$user" -c "cd ~/workspace/*/fagents-autonomy && git pull origin main" 2>/dev/null || \
    su - "$user" -c "cd ~/workspace/* && git -C .autonomy pull origin main" 2>/dev/null || true
done
```

The daemon reads `collect_telegram()` fresh each session, so restarting the agent picks up the change.

### 3. Create credential store

```bash
INFRA_USER="fagents"
INFRA_HOME=$(eval echo "~$INFRA_USER")
CLI_DIR="$INFRA_HOME/workspace/fagents-cli"

mkdir -p "$INFRA_HOME/.agents"

# Repeat for each agent that needs Telegram:
AGENT_USER="coo"           # the unix username
BOT_TOKEN="your-token-here" # from BotFather

mkdir -p "$INFRA_HOME/.agents/$AGENT_USER"
cat > "$INFRA_HOME/.agents/$AGENT_USER/telegram.env" <<EOF
TELEGRAM_BOT_TOKEN=$BOT_TOKEN
EOF

chown -R "$INFRA_USER:fagent" "$INFRA_HOME/.agents/$AGENT_USER"
chmod 700 "$INFRA_HOME/.agents/$AGENT_USER"
chmod 600 "$INFRA_HOME/.agents/$AGENT_USER/telegram.env"
```

### 4. Create sudoers rules

Skip this for agents that already have full sudo (bootstrap/ops agents).

```bash
# For each non-sudo agent:
AGENT_USER="coo"
echo "$AGENT_USER ALL=($INFRA_USER) NOPASSWD: $CLI_DIR/telegram.sh" > "/etc/sudoers.d/${AGENT_USER}-telegram"
chmod 440 "/etc/sudoers.d/${AGENT_USER}-telegram"
```

### 5. Verify

As the agent user:

```bash
# Should return a JSON error about connection/token — proves sudo works
su - coo -c "sudo -u fagents /home/fagents/workspace/fagents-cli/telegram.sh whoami"
```

With a real token, this returns `{"id":...,"is_bot":true,"username":"..."}`.

### 6. Tell the agents

Append to each agent's `MEMORY.md`:

```markdown
## Telegram
- You have Telegram via `sudo -u fagents /home/fagents/workspace/fagents-cli/telegram.sh`
- Commands: `whoami`, `send <chat-id> <message>`, `poll`
- The daemon collects incoming DMs automatically via `collect_telegram()`
- Use `send` to reply — the chat ID comes from the inbox message
- Do NOT try to access bot tokens directly — credential isolation via sudo
```

## How it works

1. A human sends a DM to the agent's bot on Telegram
2. The daemon calls `collect_telegram()` every 5 seconds
3. `collect_telegram()` runs `sudo -u fagents telegram.sh poll`
4. `telegram.sh` calls `getUpdates`, filters to text messages, outputs JSON lines
5. The daemon writes each message as a `.jsonl` file in `.queue/inbox/`
6. The agent wakes up, reads the inbox, and can reply with `telegram.sh send <chat-id> "message"`

The agent never sees the bot token. `telegram.sh` resolves credentials from `$SUDO_USER` — when coo runs `sudo -u fagents telegram.sh`, the script reads `/home/fagents/.agents/coo/telegram.env`.

## Cleanup

To remove Telegram from an agent:

```bash
rm -rf /home/fagents/.agents/coo
rm -f /etc/sudoers.d/coo-telegram
```

To remove fagents-cli entirely:

```bash
rm -rf /home/fagents/workspace/fagents-cli
rm -rf /home/fagents/repos/fagents-cli.git
```
