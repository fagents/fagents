## What changed

Comms data (channel logs, tokens, subscriptions, config, profiles, read markers, uploads) moves from inside the fagents-comms git working copy to `/home/fagents/.agents/comms/`. Separates code from data — cleaner for updates, backups, and future SQLite migration.

- **fagents-comms** — `server.py` reads/writes all data from a configurable `DATA_DIR` (via `--data-dir` flag or `DATA_DIR` env var). Falls back to the script directory for backwards compatibility.
- **fagents-autonomy** — `health-check.sh` writes alerts to the new channels path.
- **fagents** — installer passes `--data-dir` to all `server.py` calls.

## Prerequisites

None.

## Setup steps

All commands run as root on the fagents host. **Stop comms before migrating.**

```bash
INFRA_USER="fagents"
INFRA_HOME=$(eval echo "~$INFRA_USER")
COMMS_DIR="$INFRA_HOME/workspace/fagents-comms"
DATA_DIR="$INFRA_HOME/.agents/comms"
```

### 1. Pull repos

```bash
for repo in fagents-comms fagents-autonomy fagents; do
    sudo git -C "$INFRA_HOME/repos/${repo}.git" fetch "https://github.com/fagents/${repo}.git" main:main
    sudo git -C "$INFRA_HOME/workspace/${repo}" pull
done
```

### 2. Stop comms server

```bash
sudo "$INFRA_HOME/team/stop-comms.sh"
```

### 3. Move data files

```bash
mkdir -p "$DATA_DIR/channels" "$DATA_DIR/uploads"
chown -R "$INFRA_USER:fagent" "$DATA_DIR"

# Move JSON config files
for f in tokens.json channels.json subscriptions.json agent_config.json agent_health.json agent_profiles.json channel_order.json read_markers.json; do
    [ -f "$COMMS_DIR/$f" ] && mv "$COMMS_DIR/$f" "$DATA_DIR/$f" && echo "Moved $f"
done

# Move channel logs
if [ -d "$COMMS_DIR/channels" ] && [ "$(ls -A "$COMMS_DIR/channels/" 2>/dev/null)" ]; then
    mv "$COMMS_DIR/channels/"* "$DATA_DIR/channels/"
    echo "Moved channel logs"
fi

# Move uploads
if [ -d "$COMMS_DIR/uploads" ] && [ "$(ls -A "$COMMS_DIR/uploads/" 2>/dev/null)" ]; then
    mv "$COMMS_DIR/uploads/"* "$DATA_DIR/uploads/"
    echo "Moved uploads"
fi

chown -R "$INFRA_USER:fagent" "$DATA_DIR"
```

### 4. Update start-comms.sh

```bash
STOP_SCRIPT="$INFRA_HOME/team/start-comms.sh"

# Add --data-dir to the server.py serve command
if ! grep -q 'data-dir' "$STOP_SCRIPT" 2>/dev/null; then
    if command -v sed &>/dev/null && sed --version 2>/dev/null | grep -q GNU; then
        sed -i "s|server.py serve --port|server.py serve --port|;s|server.py serve|server.py serve --data-dir $DATA_DIR|" "$STOP_SCRIPT"
    else
        sed -i '' "s|server.py serve --port|server.py serve --port|;s|server.py serve|server.py serve --data-dir $DATA_DIR|" "$STOP_SCRIPT"
    fi
    echo "Updated start-comms.sh with --data-dir"
fi
```

### 5. Restart comms

```bash
sudo "$INFRA_HOME/team/start-comms.sh"
```

## How it works

Before: server.py stored all data in its own directory (the git working copy). Code and data mixed.

After: server.py stores all data in `DATA_DIR` (default: `.agents/comms/`). The git working copy is code-only. All 10 data items (8 JSON files + channels/ + uploads/) live in the data dir. Configurable via `--data-dir` flag or `DATA_DIR` env var.

## Cleanup

To revert:
```bash
# Move data back
for f in tokens.json channels.json subscriptions.json agent_config.json agent_health.json agent_profiles.json channel_order.json read_markers.json; do
    [ -f "$DATA_DIR/$f" ] && mv "$DATA_DIR/$f" "$COMMS_DIR/$f"
done
mv "$DATA_DIR/channels/"* "$COMMS_DIR/channels/" 2>/dev/null
mv "$DATA_DIR/uploads/"* "$COMMS_DIR/uploads/" 2>/dev/null

# Remove --data-dir from start-comms.sh
sed -i 's| --data-dir [^ ]*||' "$INFRA_HOME/team/start-comms.sh"

# Restart comms
sudo "$INFRA_HOME/team/stop-comms.sh" && sudo "$INFRA_HOME/team/start-comms.sh"
```
