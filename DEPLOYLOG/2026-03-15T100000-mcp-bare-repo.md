# fagents-mcp Bare Repo Migration — Deploy to Existing Installation

**Date:** 2026-03-15
**Repos changed:** fagents (install-email.sh)

## What changed

fagents-mcp was cloned directly from GitHub with origin remote intact.
All other repos use bare repo in `repos/` (origin removed) + working copy
cloned from bare. This was a bug — now fixed for new installs.

Existing installs need to migrate to the same pattern.

## Steps

```bash
INFRA_HOME="/home/fagents"
MCP_DIR="$INFRA_HOME/workspace/fagents-mcp"

# 1. Create bare repo from the existing working copy
sudo -u fagents git clone --bare "$MCP_DIR" "$INFRA_HOME/repos/fagents-mcp.git"
sudo -u fagents git -C "$INFRA_HOME/repos/fagents-mcp.git" remote remove origin

# 2. Re-point the working copy to the bare repo (instead of GitHub)
sudo -u fagents git -C "$MCP_DIR" remote set-url origin "$INFRA_HOME/repos/fagents-mcp.git"

# 3. Verify
sudo -u fagents git -C "$MCP_DIR" remote -v
# Should show: origin  /home/fagents/repos/fagents-mcp.git (fetch/push)

sudo -u fagents git -C "$INFRA_HOME/repos/fagents-mcp.git" remote -v
# Should show nothing (no remotes)
```

## Updating after migration

Same as all other repos:

```bash
# Fetch from GitHub into bare repo
sudo git -C "$INFRA_HOME/repos/fagents-mcp.git" fetch "https://github.com/fagents/fagents-mcp.git" main:main

# Pull into working copy
sudo -u fagents bash -c "cd ~/workspace/fagents-mcp && git pull"

# Rebuild + restart
sudo -u fagents bash -c "cd ~/workspace/fagents-mcp && npm run build"
sudo systemctl restart fagents-mcp
```
