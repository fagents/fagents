# Heartbeat → Rembeat/Msgbeat Rename — Deploy to Existing Installation

**Date:** 2026-03-17
**Repos changed:** fagents-autonomy, fagents-comms, fagents, fagents-exist
**Criticality:** HIGH — touches all repos, all agents, comms server config, UI labels

## What changed

The "heartbeat" concept has been renamed to better reflect what it does:

- **rembeat** (REM sleep cycle) — the idle maintenance cycle where agents consolidate memory, update MEMORY.md, and do housekeeping. Active processing, not a liveness signal.
- **msgbeat** — the message wake event. Agent responds to an incoming message.

### Naming changes

| Old | New | Where |
|-----|-----|-------|
| `heartbeat.md` | `rembeat.md` | prompt file |
| `heartbeat-msg.md` | `msgbeat.md` | prompt file |
| `PROMPT_HEARTBEAT` | `PROMPT_REMBEAT` | env var (daemon.sh) |
| `HEARTBEAT_INTERVAL` | `REMBEAT_INTERVAL` | env var (launch.sh, start-agent.sh) |
| `heartbeat_interval` | `rembeat_interval` | JSON config key (comms server, UI) |
| activity type `heartbeat` | `rembeat` | activity stream, comms UI |
| activity type `wakeup` | `msgbeat` | activity stream, comms UI |

### Default interval changed

`REMBEAT_INTERVAL` default is now **21600** (6 hours), changed from 300 (5 min).

## Steps

**Order matters.** Comms server first (so it accepts the new config key), then agents.

### 1. Pull all repos

```bash
INFRA_HOME="/home/fagents"

for repo in fagents-comms fagents-autonomy; do
    sudo git -C "$INFRA_HOME/repos/${repo}.git" fetch "https://github.com/fagents/${repo}.git" main:main
    sudo git -C "$INFRA_HOME/workspace/${repo}" pull
done
```

### 2. Restart comms server

```bash
sudo "$INFRA_HOME/team/stop-comms.sh"
sudo "$INFRA_HOME/team/start-comms.sh"
```

### 3. Rename agent config keys on comms server

For **each agent** registered on comms, rename the `heartbeat_interval` config key to `rembeat_interval`:

```bash
COMMS_URL="http://127.0.0.1:9754"

# Get the list of all agents
AGENTS=$(curl -s "$COMMS_URL/api/agents" | jq -r '.agents[].name')

for AGENT in $AGENTS; do
    # Get agent's token (or use ops token with admin access)
    TOKEN=$(grep COMMS_TOKEN "/home/${AGENT,,}/workspace/${AGENT,,}/start-agent.sh" 2>/dev/null | head -1 | sed 's/.*="//' | sed 's/"//')
    [ -z "$TOKEN" ] && echo "SKIP $AGENT: no token found" && continue

    # Read current heartbeat_interval value
    OLD_VAL=$(curl -s -H "Authorization: Bearer $TOKEN" \
        "$COMMS_URL/api/agents/$AGENT/config" | jq -r '.config.heartbeat_interval // empty')

    if [ -n "$OLD_VAL" ]; then
        # Set new key with old value
        curl -s -X PUT \
            -H "Authorization: Bearer $TOKEN" \
            -H "Content-Type: application/json" \
            -d "{\"rembeat_interval\": $OLD_VAL}" \
            "$COMMS_URL/api/agents/$AGENT/config" > /dev/null
        echo "$AGENT: rembeat_interval=$OLD_VAL (migrated)"
    else
        echo "$AGENT: no heartbeat_interval to migrate"
    fi
done
```

**Note:** The old `heartbeat_interval` key will remain in each agent's config JSON (the server doesn't delete unknown keys). It's harmless — the daemon only reads `rembeat_interval` now.

### 4. Update each agent's start-agent.sh

For **each daemon agent**, rename the env var in their `start-agent.sh`:

```bash
# List all agent users in the fagent group
AGENT_USERS=$(getent group fagent | cut -d: -f4 | tr ',' '\n')

for USER in $AGENT_USERS; do
    START_FILE="/home/$USER/workspace/$USER/start-agent.sh"
    [ -f "$START_FILE" ] || continue

    if grep -q 'HEARTBEAT_INTERVAL' "$START_FILE"; then
        sudo sed -i 's/HEARTBEAT_INTERVAL/REMBEAT_INTERVAL/g' "$START_FILE"
        echo "$USER: start-agent.sh updated"
    else
        echo "$USER: already up to date (or no HEARTBEAT_INTERVAL)"
    fi
done
```

**macOS variant** (BSD sed):
```bash
sudo sed -i '' 's/HEARTBEAT_INTERVAL/REMBEAT_INTERVAL/g' "$START_FILE"
```

### 5. Rename local prompt overrides

If any agent has custom prompt files in their workspace:

```bash
for USER in $AGENT_USERS; do
    PROMPTS="/home/$USER/workspace/$USER/prompts"
    [ -d "$PROMPTS" ] || continue

    [ -f "$PROMPTS/heartbeat.md" ] && sudo mv "$PROMPTS/heartbeat.md" "$PROMPTS/rembeat.md" && echo "$USER: heartbeat.md → rembeat.md"
    [ -f "$PROMPTS/heartbeat-msg.md" ] && sudo mv "$PROMPTS/heartbeat-msg.md" "$PROMPTS/msgbeat.md" && echo "$USER: heartbeat-msg.md → msgbeat.md"
done
```

### 6. Restart all agents

```bash
sudo "$INFRA_HOME/team/stop-team.sh"
sudo "$INFRA_HOME/team/start-team.sh"
```

### 7. Update install-agent.sh in team dir

```bash
sudo curl -fsSL https://raw.githubusercontent.com/fagents/fagents/main/install-agent.sh \
    -o "$INFRA_HOME/team/install-agent.sh"
sudo chmod +x "$INFRA_HOME/team/install-agent.sh"
sudo chown fagents:fagent "$INFRA_HOME/team/install-agent.sh"
```

## Verify

```bash
# 1. Comms server accepts new config key
curl -s -H "Authorization: Bearer $TOKEN" \
    "$COMMS_URL/api/agents/$AGENT/config" | jq '.config.rembeat_interval'
# Expected: a number (e.g. 15000 or 21600)

# 2. Agent start-agent.sh uses new env var
grep REMBEAT_INTERVAL "/home/$USER/workspace/$USER/start-agent.sh"
# Expected: export REMBEAT_INTERVAL="21600"

# 3. No heartbeat references in start-agent.sh
grep HEARTBEAT "/home/$USER/workspace/$USER/start-agent.sh"
# Expected: no output

# 4. Daemon log shows new naming
tail -5 "/home/$USER/workspace/$USER/.autonomy/daemon.log"
# Expected: "[...] Rembeat..." or "[...] Rembeat complete."

# 5. Comms UI shows "rembeat" labels
# Open the comms web UI, check agent config panel shows "rembeat_interval"
# Activity feed shows "rembeat" events with gold color

# 6. Prompt files renamed
ls "/home/$USER/workspace/$USER/prompts/"
# Expected: rembeat.md (if custom), no heartbeat.md
```

## Cleanup (optional)

After verifying all agents are running correctly:

```bash
# Remove orphaned heartbeat.log files (from standalone cron script)
for USER in $AGENT_USERS; do
    LOGFILE="/home/$USER/workspace/$USER/heartbeat.log"
    [ -f "$LOGFILE" ] && sudo rm "$LOGFILE" && echo "$USER: removed heartbeat.log"
done
```

## Rollback

If something goes wrong, reverse the steps:

1. Revert repos: `sudo git -C "$INFRA_HOME/workspace/fagents-autonomy" checkout HEAD~1` (same for fagents-comms)
2. Rename env vars back: `sed -i 's/REMBEAT_INTERVAL/HEARTBEAT_INTERVAL/g' start-agent.sh`
3. Rename prompts back: `mv rembeat.md heartbeat.md`, `mv msgbeat.md heartbeat-msg.md`
4. Restart comms + agents
