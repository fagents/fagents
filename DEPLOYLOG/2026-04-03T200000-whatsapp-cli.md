# WhatsApp CLI — Deploy to Existing Installation

**Date:** 2026-04-03
**Repos changed:** fagents-cli, fagents-autonomy, fagents

## What changed

- **fagents-cli** — new `whatsapp.mjs`: WhatsApp client via Baileys. Self-chat model (human sends "Note to self", agent reads). Background `serve` process maintains WebSocket, flat-file IPC via spool/outbox dirs. Voice download + transcription via stt-transcribe.sh. 47 tests.
- **fagents-autonomy** — `collect_whatsapp()` in daemon.sh, `ensure_whatsapp_serve()` lifecycle, WHATSAPP_CADENCE=3.
- **fagents** — installer support (both Linux + macOS), E2E test coverage (96/96 Linux, 110/110 macOS).

## Prerequisites

- **Node.js** must be installed on the host. `whatsapp.mjs` is the first Node.js tool in fagents-cli.
  - Linux: `apt-get install nodejs npm`
  - macOS: `brew install node`
- **WhatsApp account** — the human scans a QR code to link. One account per agent (like Telegram bots). **Ask the human** to do the QR scan — agents cannot do this. Login auto-detects the self-chat JID from the linked account (no phone number needed).

## Setup steps

All commands run as root on the fagents host. Adapt variable values to your deployment.

```bash
INFRA_USER="fagents"
INFRA_HOME=$(eval echo "~$INFRA_USER")
CLI_DIR="$INFRA_HOME/workspace/fagents-cli"
AGENT_USER="<unix-username>"  # e.g. ftl — the agent that will use WhatsApp
```

### 1. Pull repos

```bash
# fagents-cli (new files: whatsapp.mjs, package.json, whatsapp/SKILL.md)
sudo git -C "$INFRA_HOME/repos/fagents-cli.git" fetch "https://github.com/fagents/fagents-cli.git" main:main
sudo git -C "$INFRA_HOME/workspace/fagents-cli" pull

# fagents-autonomy (daemon.sh changes)
sudo git -C "$INFRA_HOME/repos/fagents-autonomy.git" fetch "https://github.com/fagents/fagents-autonomy.git" main:main
sudo git -C "$INFRA_HOME/workspace/fagents-autonomy" pull
```

### 2. Install Node.js dependencies

```bash
(cd "$CLI_DIR" && sudo -u "$INFRA_USER" npm install --production)
```

Verify:

```bash
test -d "$CLI_DIR/node_modules/@whiskeysockets" && echo "ok" || echo "MISSING: baileys"
```

### 3. Link WhatsApp account (interactive — human required)

**Important:** Must run as the agent user (not root), so `SUDO_USER` resolves correctly and the session is saved to the right `.agents/` directory.

```bash
su - "$AGENT_USER" -c "sudo -u $INFRA_USER $CLI_DIR/whatsapp.mjs login"
```

A QR code appears in the terminal. The human scans it with WhatsApp on their phone (Settings > Linked Devices > Link a Device). After successful link:

```bash
su - "$AGENT_USER" -c "sudo -u $INFRA_USER $CLI_DIR/whatsapp.mjs whoami"
# Should show: {"jid":"...@s.whatsapp.net","name":"..."}
```

Session and `whatsapp.env` (with self-chat JID) are both written automatically by login. No phone number needed — login detects the JID from the linked account.

**Sequence matters:** Login must happen BEFORE the daemon restart (step 7). The daemon's `serve` process needs the session to connect.

### 4. Create spool/outbox dirs and fix permissions

```bash
mkdir -p "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-spool"
mkdir -p "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-outbox"

chown -R "$INFRA_USER:fagent" "$INFRA_HOME/.agents/$AGENT_USER"
chmod 700 "$INFRA_HOME/.agents/$AGENT_USER"
chmod 600 "$INFRA_HOME/.agents/$AGENT_USER/whatsapp.env"
```

### 5. Update sudoers rules

```bash
# Append whatsapp.mjs to existing telegram sudoers rule
if [[ -f "/etc/sudoers.d/${AGENT_USER}-telegram" ]]; then
    existing=$(cat "/etc/sudoers.d/${AGENT_USER}-telegram")
    echo "${existing}, $CLI_DIR/whatsapp.mjs" > "/etc/sudoers.d/${AGENT_USER}-telegram"
    chmod 440 "/etc/sudoers.d/${AGENT_USER}-telegram"
else
    echo "$AGENT_USER ALL=($INFRA_USER) NOPASSWD: $CLI_DIR/whatsapp.mjs" > "/etc/sudoers.d/${AGENT_USER}-whatsapp"
    chmod 440 "/etc/sudoers.d/${AGENT_USER}-whatsapp"
fi
```

### 6. Install skill

```bash
AGENT_HOME=$(eval echo "~$AGENT_USER")
mkdir -p "$AGENT_HOME/.claude/skills/whatsapp"
cp "$CLI_DIR/whatsapp/SKILL.md" "$AGENT_HOME/.claude/skills/whatsapp/SKILL.md"

# BSD sed (macOS) needs -i '', GNU sed (Linux) needs just -i
if sed --version 2>/dev/null | grep -q GNU; then
    sed -i "s|__CLI_DIR__|$CLI_DIR|g" "$AGENT_HOME/.claude/skills/whatsapp/SKILL.md"
else
    sed -i '' "s|__CLI_DIR__|$CLI_DIR|g" "$AGENT_HOME/.claude/skills/whatsapp/SKILL.md"
fi
```

### 7. Restart daemon

The daemon needs to restart to pick up `collect_whatsapp()` and start `whatsapp.mjs serve`:

```bash
# Self-restart (daemon restarts itself, no downtime gap)
sudo -u "$AGENT_USER" bash -c "cd ~/workspace/$AGENT_USER && kill \$(cat .autonomy/daemon.pid)"
# Daemon auto-restarts via systemd/launchd
```

## Verify

```bash
# Check serve is running
ps aux | grep "whatsapp.mjs serve" | grep -v grep

# Test send (to self-chat)
su - "$AGENT_USER" -c "sudo -u $INFRA_USER $CLI_DIR/whatsapp.mjs send $SELF_JID 'test from agent'"

# Have the human send a message in "Note to self", then:
su - "$AGENT_USER" -c "sudo -u $INFRA_USER $CLI_DIR/whatsapp.mjs poll"
```

## Doctor

```bash
echo "--- WhatsApp doctor ---"
test -x "$CLI_DIR/whatsapp.mjs" && echo "ok: whatsapp.mjs executable" || echo "FAIL: whatsapp.mjs not executable"
test -d "$CLI_DIR/node_modules/@whiskeysockets" && echo "ok: baileys installed" || echo "FAIL: run npm install"
test -f "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-session/creds.json" && echo "ok: session exists" || echo "FAIL: run login"
test -f "$INFRA_HOME/.agents/$AGENT_USER/whatsapp.env" && echo "ok: whatsapp.env exists" || echo "FAIL: create whatsapp.env"
test -d "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-spool" && echo "ok: spool dir exists" || echo "FAIL: mkdir whatsapp-spool"
grep -q "whatsapp.mjs" /etc/sudoers.d/${AGENT_USER}-telegram 2>/dev/null || grep -q "whatsapp.mjs" /etc/sudoers.d/${AGENT_USER}-whatsapp 2>/dev/null && echo "ok: sudoers configured" || echo "FAIL: update sudoers"
```

## Cleanup

To remove WhatsApp for an agent:

```bash
rm -rf "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-session"
rm -rf "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-spool"
rm -rf "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-outbox"
rm -f "$INFRA_HOME/.agents/$AGENT_USER/whatsapp.env"
rm -f "$INFRA_HOME/.agents/$AGENT_USER/whatsapp-serve.pid"

# Remove from sudoers — recreate without whatsapp.mjs
# (check current content first, remove the whatsapp.mjs entry)
```
