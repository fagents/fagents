**Date:** 2026-03-05
**Repos changed:** fagents-cli (new), fagents-autonomy, fagents

New installs get Telegram automatically via the installer. This doc is for adding Telegram to an existing deployment.

## What changed

- **fagents-cli** — new repo. `telegram.sh` is a bash+curl+jq Bot API client. Commands: `whoami`, `send <chat-id> <message>`, `poll`.
- **fagents-autonomy** — `collect_telegram()` in `daemon.sh` now calls `telegram.sh poll` via sudo and writes messages to the inbox queue. Was previously a stub.
- **fagents** — installer handles Telegram for new installs (credential store, sudoers, fagents-cli clone).

## Prerequisites

- One Telegram bot token per agent (from @BotFather). Each agent needs their own bot — `getUpdates` is destructive (consumes offsets), so only one poller per bot.
- Your Telegram user ID (numeric). Send `/start` to the bot, then run `curl -s "https://api.telegram.org/bot<TOKEN>/getUpdates" | jq '.result[0].message.from.id'`. This gates the bot so only you can talk to it.

## Setup steps

All commands run as root on the fagents host. Adapt variable values to your deployment.

```bash
INFRA_USER="fagents"
INFRA_HOME=$(eval echo "~$INFRA_USER")
```

### 1. Clone fagents-cli

New repo — not present on existing deployments.

```bash
su - "$INFRA_USER" -c "git clone --bare https://github.com/fagents/fagents-cli.git ~/repos/fagents-cli.git"
su - "$INFRA_USER" -c "git -C ~/repos/fagents-cli.git remote remove origin"
chmod -R g+rX "$INFRA_HOME/repos/fagents-cli.git"

su - "$INFRA_USER" -c "git clone $INFRA_HOME/repos/fagents-cli.git ~/workspace/fagents-cli"
chmod -R g+rX "$INFRA_HOME/workspace/fagents-cli"
```

### 2. Update fagents-autonomy

The shared autonomy working copy needs the new `collect_telegram()` from commit `167d31b`:

```bash
su - "$INFRA_USER" -c "cd ~/workspace/fagents-autonomy && git pull"
```

Agents pick up the change on their next daemon restart — no per-agent action needed.

### 3. Create credential store

Repeat the block inside the loop for each agent that should have Telegram.

```bash
mkdir -p "$INFRA_HOME/.agents"

# For each agent that needs Telegram:
AGENT_USER="<unix-username>"    # e.g. ftf, coo — the agent's unix user
BOT_TOKEN="<token-from-botfather>"
ALLOWED_ID="<telegram-user-id>"  # numeric ID of the human who talks to this bot (optional, recommended)

mkdir -p "$INFRA_HOME/.agents/$AGENT_USER"
cat > "$INFRA_HOME/.agents/$AGENT_USER/telegram.env" <<EOF
TELEGRAM_BOT_TOKEN=$BOT_TOKEN
TELEGRAM_ALLOWED_IDS=$ALLOWED_ID
EOF

chown -R "$INFRA_USER:fagent" "$INFRA_HOME/.agents/$AGENT_USER"
chmod 700 "$INFRA_HOME/.agents/$AGENT_USER"
chmod 600 "$INFRA_HOME/.agents/$AGENT_USER/telegram.env"
```

### 4. Create sudoers rules

For each agent that needs Telegram and does NOT already have full sudo:

```bash
CLI_DIR="$INFRA_HOME/workspace/fagents-cli"
AGENT_USER="<unix-username>"

echo "$AGENT_USER ALL=($INFRA_USER) NOPASSWD: $CLI_DIR/telegram.sh" > "/etc/sudoers.d/${AGENT_USER}-telegram"
chmod 440 "/etc/sudoers.d/${AGENT_USER}-telegram"
```

Agents with full sudo (bootstrap/ops) already have access — skip them.

### 5. Verify

For each agent with Telegram, test that the sudo path works:

```bash
su - <agent-user> -c "sudo -u fagents $INFRA_HOME/workspace/fagents-cli/telegram.sh whoami"
```

With a dummy/invalid token this returns `{"error":"..."}` — that's correct, it proves the sudo + credential chain works. With a real token it returns `{"id":...,"is_bot":true,"username":"..."}`.

### 6. Update agent MEMORY.md

For each agent with Telegram, append to their workspace's `memory/MEMORY.md`:

```markdown
## Telegram
- You have Telegram via `sudo -u fagents /home/fagents/workspace/fagents-cli/telegram.sh`
- Commands: `whoami`, `send <chat-id> <message>`, `poll`
- The daemon collects incoming DMs automatically via `collect_telegram()`
- Use `send` to reply — the chat ID comes from the inbox message
- Do NOT try to access bot tokens directly — credential isolation via sudo
```

## How it works

1. A human sends a DM to the agent's bot on Telegram
2. The daemon calls `collect_telegram()` every 5 seconds
3. `collect_telegram()` runs `sudo -u fagents telegram.sh poll`
4. `telegram.sh` resolves the caller from `$SUDO_USER`, reads `/home/fagents/.agents/<caller>/telegram.env`
5. Calls `getUpdates`, filters to text messages, gates by `TELEGRAM_ALLOWED_IDS` if set, outputs one JSON line per message
6. The daemon writes each message as a `.jsonl` file in `.queue/inbox/`
7. The agent wakes, reads the inbox, and can reply with `telegram.sh send <chat-id> "message"`

The agent never sees the bot token.

## Cleanup

To remove Telegram from a single agent:

```bash
rm -rf $INFRA_HOME/.agents/<agent-user>
rm -f /etc/sudoers.d/<agent-user>-telegram
```

To remove fagents-cli entirely:

```bash
rm -rf $INFRA_HOME/workspace/fagents-cli
rm -rf $INFRA_HOME/repos/fagents-cli.git
```
